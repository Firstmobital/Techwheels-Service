# Implementation Plan Automation Instructions

## Automatic Plan Creation

**When you ask for an implementation plan, I will automatically:**

1. ✅ Create a new plan file in `docs/Implementation_plans/[PLAN_NAME].md`
2. ✅ Fill in all sections with complete, actionable content (no placeholders left)
3. ✅ Set up activity tracker with real-time status indicators
4. ✅ Update `docs/Implementation_plans/INDEX.md` with the new plan
5. ✅ Organize tasks into logical phases
6. ✅ Include risk assessment, success criteria, and detailed breakdowns

**No manual template copying or editing needed.**

## How to Request a Plan

Use natural language. No special format required. Examples:

- "Create an implementation plan for mobile app push notifications"
- "I need a plan to migrate the database to PostgreSQL"
- "Set up a plan for user analytics tracking"
- "Create a plan to implement dark mode support"

## Activity Tracker Updates

The activity tracker in each plan uses these symbols:
- ⏳ PENDING — Task not started
- 🔄 IN PROGRESS — Currently working
- ✅ COMPLETED — Finished
- ❌ BLOCKED — Waiting on external dependency

Update these in the plan file as work progresses. I will ask for updates or check status upon request.

## Plan File Structure

All plans include:
- Executive Summary with risk level & duration
- Objectives & context
- Implementation tasks organized into phases
- Real-time activity tracker
- Detailed task breakdowns with acceptance criteria
- Risk assessment matrix
- Success criteria & sign-off section
- Notes & lessons learned section

## Dashboard & Navigation

- **Quick Overview:** Check `docs/Implementation_plans/INDEX.md` for all active/completed plans
- **Full Details:** Click on individual plan file for complete information
- **Progress Updates:** Edit activity tracker in each plan file as work advances

## No Reminders Needed

You don't need to remind me of this. Any request for a plan will trigger automatic creation with full structure and content. Just ask naturally.
