# AutoDoc Category

**Purpose:** AutoDoc feature development, testing, and operational documentation.

## Structure

- `active/`: Active AutoDoc implementation plans and feature specs

- `evidence/`: Execution status, feature maps, test reports
  - Feature implementation status and checklists
  - Format validation test reports
  - Architecture and design decisions

- `runbooks/`: Testing procedures and operational guides
  - Test execution guides
  - Feature testing procedures
  - Deployment checklists

## Related Docs

- Root-level evidence: [docs/autodoc/evidence/](../../autodoc/evidence/)
- Root-level runbooks: [docs/autodoc/runbooks/](../../autodoc/runbooks/)

## Adding New Docs to This Category

- New AutoDoc feature plan? → `active/`
- Test report or execution status? → `evidence/`
- Testing guide or procedure? → `runbooks/`
