Report 3 — Payment Status — All Categories

💳
Payment status — all warranty categories
wc · updation · amc · goodwill · fsb · claim_settlement — per-table DB-exact counts
Per-table logic — which field = terminal state
📋WC / Updation / AMC / Goodwill / Part WC: use source_row_data->>'claim_status' directly. "Settled" = TM paid.
📋FSB (Free Service Billing): uses claim_status. "Accepted" = TM accepted and will settle (terminal success). "Rejected" = terminal failure. NOT normalised to awaiting_sop.
📋Settlement Register: has no claim_status. Uses posting_document_number. Empty string = SAP not posted yet (unposted). Non-empty = settled in SAP.
⚠️Critical bug in existing code: normalizeStatusBucket() maps "Accepted" → awaiting_sop. Correct mapping: "Accepted" = Settled/Accepted by TM (success state). This inflated SOP count and deflated settled count across FSB + Goodwill rows.
