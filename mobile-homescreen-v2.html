Design Philosophy

Platform-first, not feature-first
Like Amazon / Tata Motors app — Home is a launcher, not a screen. Every module lives as a tile. New modules just get added to the grid.
Scale without redesign
LIVE / SOON / NEW badges let you ship future modules (Body Shop, Mechanical, Warranty, Parts) without rebuilding the home screen.

Zone Breakdown

① Header
Logo · Greeting · Notification bell (with red dot) · Profile avatar. Dark-blue gradient feels premium.
② Search Bar
Search job cards by reg no., vehicle, or report name. Barcode/QR scan icon for fast intake lookup.
③ Stats Strip (3 numbers)
Active Jobs · Pending Action · Done This Month. Floating card that pops off the header.
④ Service Modules Grid
3-column icon tiles. One tap to any module. Status badges signal readiness.
⑤ Recent Activity Feed
Cross-module feed — any action from any module shows here. Like a notification log on screen.
⑥ Tab Bar (5 tabs)
Home · Search · + New (floating centre CTA) · Alerts · Profile. Clean and universal.

Module Tiles Planned

🎨 Body & Paint 📊 Reports 📥 Import 👨‍💼 Admin 🏭 Body Shop ⚙️ Mechanical 🛠️ Service Bay 🔩 Parts & Stock 🛡️ Warranty

What Changed from v1

✗ v1 was AutoDoc-centric — KPIs only showed job cards
✗ v1 had no logo / brand identity
✗ v1 had no search / QR scan
✗ v1's tab bar had domain tabs (Import, Reports) mixed with platform nav

✓ v2 is a module launcher
✓ Notification bell + profile avatar (standard mobile UX)
✓ Search bar with QR scan shortcut
✓ Coming-soon tiles = roadmap visible in the app
✓ Tab bar = platform nav (Home / Search / New / Alerts / Profile)
✓ Cross-module recent activity feed
