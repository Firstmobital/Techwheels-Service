FSB business logic — different from WC
🔑FSB uses claim_status field, not sr_status. Values: "Accepted" (TM approved = terminal success), "Rejected" (terminal failure), "Created" (pending).
🔑FSB tracks Free Services (1st, 2nd, 3rd, 4th) via service_type numeric code (1/2/3/4).
⚠️Critical frontend bug: existing code maps "Accepted" → awaiting_sop bucket. Correct: "Accepted" = TM accepted and will pay the dealer. This is a SUCCESS state.
⚠️sr_status field only shows "Closed/Open/Cancel" (the service request status, not claim approval). Do NOT use sr_status for KPI computation.
